Methods

4 phương pháp đang so sánh trên hệ full

Frontend exposed
MethodVai tròKhi dùng
es_bm25Lexical baseline nhanh nhất, search title/content trong Elasticsearch.Fallback low-latency, sanity check, entity overlap mạnh.
tv_denseDense-only retrieval qua TurboVec, hydrate docs từ Elasticsearch.Đo signal semantic độc lập của dense index.
tv_hybridBM25 candidates + TurboVec candidates fused bằng Reciprocal Rank Fusion.Default demo vì quality tốt nhất trong pilot.
tv_filtered_hybridDùng BM25 numeric_id allowlist để constrain dense search rồi fuse.So sánh speed/quality, chưa thay default do mất recall.

Sprint 3 đã đạt mục tiêu engineering chính: dense retrieval full-corpus không còn bị khóa trong Elasticsearch vector 5k docs. TurboVec đã thay dense backend cho 5.23M docs, frontend/backend đã nối vào runtime hiện tại, và benchmark page đã phản ánh 4 method mới.

Recommendation

Dùng tv_hybrid làm default demo. Giữ es_bm25 làm fallback nhanh, dùng tv_filtered_hybrid để nghiên cứu speed/recall trade-off chứ chưa thay default.
